STEP 5 — reportDate
----------------------------------------
Source column: dataFilter/reportDate.txt (flat If/then/else chain, not
                nested — 6 sequential conditions, each returning another
                column's value rather than a literal)
Keyed on:       productNew (Step 2), Voided, DateOfCreated, VoidDate,
                DATEOFSERVICESTART — must run AFTER Step 2 (needs
                productNew)
Logic:          non-SPECIALIST rows use DateOfCreated (or VoidDate if
                voided); SPECIALIST rows use DateOfCreated/VoidDate only
                if it's later than DATEOFSERVICESTART, otherwise they use
                DATEOFSERVICESTART itself
Assumption:     the rule file compares "Voided = 1" for the voided case,
                but this dataset encodes voided rows as Voided = -1 (a
                legacy true/false convention), not 1. Treated "Voided = 1"
                as "Voided <> 0" so the voided branch still fires
                correctly — confirmed against the 3 voided rows, which
                all correctly picked up VoidDate as reportDate.
Default:        none — the 6 conditions are exhaustive over
                (productNew = SPECIALIST or not) x (voided or not) x
                (date-vs-DATEOFSERVICESTART); 0 nulls in this dataset.

STEP 7 — output_active.xlsx / output_void.xlsx split
----------------------------------------
Source file: output/output.xlsx (589 rows, 132 columns)

  - added column voidStatus, filled "all" for every row
  - 589 rows x 133 columns

output_void.xlsx:

    SUBTOTAL, SUBTOTALORG, TAX, VAT
  - 3 rows x 133 columns

All 4 files are written via write_excel_overwrite() (pipeline.py): if a
file already exists at the target path it's deleted first, then the new
one is written — a re-run always replaces stale output outright, never
merges/appends, and fails with a clear "close it in Excel" error instead
of a bare PermissionError if the target is locked open.

This run was triggered from the CLI (`python pipeline.py rawData/original.xlsx`).
The same 8 steps run identically from the Streamlit "Run Pipeline" page
and from an unattended Windows Task Scheduler job configured on the
